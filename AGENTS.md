# Project Instructions

## Phase 5 Changelog Discipline

- For every meaningful Phase 5 project update, append or update
  `docs/phase-5-changelog.md`.
- Record implementation changes, ERAWAN runs, diagnostics, benchmark results,
  conclusions, artifact-retention decisions, and next steps.
- Treat `docs/phase-5-changelog.md` as the report source of truth, so future
  Codex chats can reconstruct the project history without relying on chat logs.
- When a new chat resumes this project, inspect `docs/project-state.md`,
  `docs/phase-5-master-plan.md`, `docs/phase-5-erawan-runbook.md`, and
  `docs/phase-5-changelog.md` before continuing Phase 5 work.
